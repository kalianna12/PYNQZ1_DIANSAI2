# MAX5885 + LTC2208 Integration

## Fixed architecture

- Board: Lemon PYNQ-Z1 compatible board, XC7Z020.
- MAX5885 dual DAC: unchanged, 200 MSPS, left P1 2x17 data header.
- LTC2208 dual ADC: 130 MSPS, right P2 2x17 data header.
- PS control: AXI-Lite registers and AXI DMA through PYNQ Python. No PL UART is present.
- Storage path: LTC2208 -> 64-bit asynchronous AXIS FIFO -> AXI DMA S2MM -> PS DDR.

The ADC does not use an AD9226 compatibility layout and does not require any
jumper wiring. Treat P1 and P2 as two independent 2x17 female data headers.
Power and ground are connected separately. Every LTC2208 data bit below is
directly wired to the named P2 FPGA pin.

`ltc_a_clk` and `ltc_b_clk` are generated by the FPGA at 130 MHz. They are
the ADC sampling clocks. ADC output buses are captured by a companion 130 MHz
clock phase generated from the same MMCM, with IOB input registers.

## PS address map

| IP | Base | Range | Use |
|---|---:|---:|---|
| `led_ctrl_0` | `0x40000000` | `0x1000` | board LEDs, RGB LEDs, buttons |
| `max5885_ctrl_0` | `0x40001000` | `0x1000` | all MAX5885 signal parameters |
| `ltc2208_capture_0` | `0x40002000` | `0x1000` | LTC2208 acquisition parameters/status |
| `axi_dma_0` | `0x40400000` | AXI DMA | S2MM samples to DDR |

## LTC2208 capture contract

- Sample rate: 130,000,000 dual-channel samples/s.
- Sample word in DDR: `uint32`, bits `[15:0] = A`, bits `[31:16] = B`.
- Samples are offset-binary. For signed plotting use `code - 0x8000`.
- `channel_mask`: `0b01` A, `0b10` B, `0b11` both.
- `decimation=1` is full 130 MSPS. It is exposed for debug but not needed for full-rate acquisition.
- Verified capture maximum is 262,144 dual-channel samples. The stream has no
  BRAM sample buffer; the limit is a deliberate validated DMA software limit,
  and can later be raised only together with a DMA BTT validation run.
- 65,536 samples last about 0.504 ms. One million samples last about 7.692 ms.

## Clock-domain and timing design

- AXI control and DMA memory side: 125 MHz PS FCLK.
- MAX5885 sample domain: 200 MHz.
- LTC2208 clock forward domain: 130 MHz.
- LTC2208 capture and AXIS FIFO write side: 130 MHz.
- AXIS FIFO read side and DMA: 125 MHz.

The 125 MHz control domain and 130 MHz ADC domains are explicitly asynchronous.
The start command uses a synchronized toggle; configuration is written before
the toggle and held constant for the acquisition. Status returns through
synchronizers. Sample data crosses only through the asynchronous AXIS FIFO.

The final Vivado 2022.1 routed implementation is accepted only with positive
WNS. The current implementation report records WNS `+0.007 ns` at 200 MHz and
no failing setup endpoints.
